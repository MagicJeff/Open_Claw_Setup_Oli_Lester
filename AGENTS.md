# AGENTS.md

This public snapshot should stay clean, factual, and easy to scan.

## Priorities

- optimise for technical clarity over breadth
- prefer architecture and trade-offs over hype
- keep claims conservative and defensible
- do not reintroduce private runtime state from the main repo

## What To Update

- `README.md` for the high-level portfolio narrative
- `docs/architecture.md` when the architecture story changes
- `docs/decisions.md` when a design trade-off changes
- `docs/use-cases.md` when a public-safe workflow should be added or removed
- `src/contracts/` only for representative, non-sensitive code patterns

## What Not To Add

- secrets, tokens, session files, or credentials
- personal memories, inbox data, CV drafts, or personal job-search records
- vague marketing copy
- claims that are not visible in the parent working repository

## Writing Rules

- use British English
- write for a technical hiring manager
- prefer short direct sentences
- avoid startup-style slogans and buzzwords
